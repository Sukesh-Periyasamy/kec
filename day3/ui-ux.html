Flexbox ‚Äì Real Usage

Flexbox = One-Dimensional Layout System

Controls items in a single axis ‚Äî either a row or a column. Perfect for navbars, card rows, and centering content.

Navigation bars Card alignment Centering content

Basic Example

.container { display: flex; justify-content: space-between; align-items: center; }

Common Properties

Property Purpose
justify-content Main-axis alignment
align-items Cross-axis alignment
flex-direction Row or column
gap Spacing between items
flex-wrap Wrap to next line

üî• Real Portfolio Examples

Navbar:

nav { display: flex; justify-content: space-between; align-items: center; }

Cards section:

.cards { display: flex; gap: 20px; flex-wrap: wrap; }

CSS Grid ‚Äì Real Usage

Grid = Two-Dimensional Layout System

Controls items across both rows and columns simultaneously. Ideal for dashboards, complex page layouts, and gallery sections.

Dashboards Complex layouts Gallery sections

Basic Example

.container { display: grid; grid-template-columns: repeat(3, 1fr); gap: 20px; }

Template Areas

grid-template-areas: "header header" "sidebar content" "footer footer";

Define exact layout zones by name ‚Äî extremely powerful for page structure.

üì¶ Flexbox
  • 1D ‚Äî one axis at a time
  • Best for components & sections
  • Navbars, card rows, centering
  • Easy alignment control
üß© CSS Grid
  • 2D ‚Äî rows AND columns
  • Best for full page layouts
  • Dashboards, galleries, templates
  • Advanced named areas

Rule of Thumb

Flexbox ‚Üí Small sections & component alignment  ¬∑  Grid ‚Üí Full page layout & complex structures

Accessibility Basics

Making websites usable for everyone

Including: visually impaired users, keyboard-only users, and screen reader users. Accessibility is a professional standard and is required in many companies.

1
Use Semantic HTML ‚Äî use meaningful elements so screen readers understand the page structure:
<header> <nav> <main> <section> <footer> /* Not just <div> everywhere */
2
Add Alt Text to Images ‚Äî describe what the image shows:
<img src="profile.jpg" alt="Sukesh profile photo">
3
Good Color Contrast ‚Äî avoid light gray text on white background. Use contrast checker tools. Aim for WCAG AA ratio (4.5:1 for normal text).
4
Keyboard Navigation ‚Äî users should be able to navigate using Tab key, and every interactive element must show a visible focus outline.

üî• Why Accessibility Matters

‚úÖ Professional standard
‚úÖ Required at many companies
‚úÖ Improves SEO ranking
‚úÖ Inclusive design for all

Component Thinking

‚ùå Beginner Thinking
  • "I'm designing a page"
  • Write each card HTML manually
  • Unique styles per element
  • Hard to maintain or update
‚úÖ Developer Thinking
  • "I'm building reusable components"
  • Define one card, reuse everywhere
  • Shared design system
  • Scalable & easy to update

üîπ What is a Component?

Small, self-contained reusable blocks:

Navbar Card Button Modal Footer

Example ‚Äî one Card component, used everywhere:

<div class="card"> <h3>Project Title</h3> <p>Description</p> </div> /* Style once. Reuse everywhere. */

üîπ Why It Matters

Modern frameworks like React and Vue are entirely built on this concept. Thinking in components:

  • Improves code structure and readability
  • Makes your UI reusable across pages
  • Makes scaling the project much easier
  • Prepares you for React & Vue development

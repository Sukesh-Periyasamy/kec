Environment Configuration (.env)

üß† What is an Environment Variable?

Store sensitive or configurable values outside your code so they never end up in your repo.

‚ùå Never do this

‚úÖ Do this instead

üì¶ Why It's Important

üîí Security ‚Äî no passwords on GitHub
üîÑ Different configs for dev & prod
üöÄ Easy & clean deployment

üìÅ Example .env File

üîß Using in Node.js

Install dotenv:

Use in your app:

üîê Production Tip ‚Äî NEVER push .env to GitHub!

Add to .gitignore:

On AWS/production ‚Äî set vars manually:

Build Mode vs Development Mode

üõ†Ô∏è Development Mode
  • Used while actively coding
  • Fast hot-reload on save
  • Full debug tools & error overlays
  • Source maps for debugging
  • Unoptimized, large file sizes
  • Run: npm start
üöÄ Production Build
  • Used for deployment only
  • Minified & compressed code
  • Tree shaking removes dead code
  • Faster loading times
  • Optimized chunk splitting
  • Run: npm run build

üî• Why Important?

Without production build ‚ùå

  • Website is slow
  • Large file sizes
  • Poor Lighthouse score

With production build ‚úÖ

  • Faster load times
  • Smaller bundle size
  • High performance score

npm run build generates a /build folder ‚Äî deploy only this folder.

SEO Basics

SEO = Making your site visible on Google

If your portfolio has no SEO, recruiters may not find you and your Google ranking stays low. These are the fundamentals every developer must know.

1
Title Tag ‚Äî the page title shown in Google results and browser tabs:
<title>Sukesh | Full Stack Developer</title>
2
Meta Description ‚Äî snippet shown under your page title in search results:
<meta name="description" content="Full stack developer skilled in Node.js, MongoDB and AWS.">
3
Proper Heading Hierarchy ‚Äî never skip levels:
<h1>Main Title</h1> ‚Üê one per page <h2>Section</h2> <h3>Subsection</h3>
4
Semantic HTML ‚Äî use <header>, <nav>, <main>, <section>, <footer> ‚Äî not every element as a <div>.
5
Clean URLs ‚Äî human-readable paths help both users and Google:
/portfolio   /projects   /contact
/page?id=123&ref=xyz ‚Üê bad
6
Sitemap (Advanced) ‚Äî create a sitemap.xml file and submit it to Google Search Console so all your pages get indexed.

Performance Optimization

Performance affects everything

User Experience SEO Ranking Bounce Rate Conversion Rate

üìâ Reduce HTTP Requests

Combine CSS & JS files when possible. Every extra request adds latency. Bundle your assets using Webpack or Vite.

üóúÔ∏è Enable Server Compression

üåç Use a CDN

üò¥ Lazy Loading Images

Image Optimization

Images are the #1 reason websites slow down

Unoptimized images can account for 60‚Äì80% of total page weight.

üîπ Use the Correct Format

Format Best For Notes
JPEG Photos & complex images Good quality/size tradeoff
PNG Transparent images Larger file size
WebP Modern compressed format ‚≠ê Best choice ‚Äî 30% smaller than JPEG
SVG Icons & logos Vector, scales perfectly
üëâ Best modern choice: WebP ‚Äî same visual quality, ~30% smaller file size

üóúÔ∏è Compress Images

  • Use TinyPNG (web tool)
  • Use ImageOptim (Mac app)
  • Use Squoosh (Google, web)
2 MB ‚Üí 200 KB (90% saved)

üìê Resize Before Upload

Don't upload a 4000√ó3000px image if displaying it at 400px.

/* Correct ‚Äî resize first */ width: 400px; ‚Üí upload 400px file /* Wrong */ width: 400px; ‚Üí upload 4000px file

üîπ Responsive Images ‚Äî Browser Loads the Right Size

The browser automatically picks the smallest image that looks good on the user's screen.

üî• Bonus: Browser Caching (Nginx)

Store static files in the browser ‚Äî faster repeat visits:

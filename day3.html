What is Version Control?

Real-Life Example

Imagine writing your final year project report:

  • You edit it 10 times
  • Suddenly your file gets corrupted
  • How will you recover the previous version?

That's where Version Control System (VCS) helps!

Definition

Version control tracks changes in code and allows you to revert to previous versions.

Why Industry Uses Git?

  • Team Collaboration: Multiple developers working simultaneously
  • Tracking Changes: Complete history of modifications
  • Undo Mistakes: Rollback to previous versions
  • Code Review: Review before merging
  • Deployment Automation: CI/CD pipelines
  • Internship Requirement: GitHub profile matters!

Your GitHub profile is your technical resume.

What is Git?

Git is a distributed version control system

Developed By

Linus Torvalds (Creator of Linux)

Used In

  • Startups
  • MNCs (Google, Microsoft, Meta)
  • Open-source projects
  • Research labs

Difference Between Git and GitHub

Git

  • Software/Tool
  • Works locally on your computer
  • Tracks code changes
  • Command-line tool

GitHub

  • Online platform/Service
  • Works in the cloud
  • Stores repositories online
  • Web-based interface

Other Similar Platforms:

GitLab • Bitbucket

Repository Concept

Repository = Project folder tracked by Git

Two Types:

Local Repository

On your computer

Remote Repository

On GitHub cloud

Local Laptop
GitHub Cloud

PART 2 – Basic Git Commands

Live Demo Time!

Step 1 – Git Installation Check

git --version

If not installed: Download from git-scm.com

Step 2 – Configure Git (VERY IMPORTANT)

git config --global user.name "Your Name"
git config --global user.email "your@email.com"

This identifies who made the commit!

Git Commands (Continued)

Step 3 – Initialize Repository

git init

Creates hidden .git folder

Step 4 – Add Files (Staging)

git add .

Adds files to staging area

Step 5 – Commit (Save Point)

git commit -m "Initial commit"

Commit = Checkpoint/Save point

Connecting to GitHub

Step 6 – Connect to GitHub

git remote add origin <repo_link>

Step 7 – Push to GitHub

git push -u origin main

Uploads your code to GitHub cloud!

Step 8 – Clone Repository

git clone <repo_link>

Real-life scenario: If you join a company → First thing you do is clone the project

PART 3 – Branching Concept

Industry Workflow

Why Branching?

  • main branch = Stable production version
  • feature branch = Development work

Example Structure:

main
   |
   |--- feature-login
   |
   |--- feature-ui

Commands:

git branch feature-login
git checkout feature-login
git merge feature-login

Never work directly on main in a company!

PART 4 – Deployment Using GitHub Pages

Step 1: Push portfolio to GitHub
git add .
git commit -m "Portfolio ready"
git push
⬇️
Step 2: Enable GitHub Pages
  • Go to Repository
  • Settings → Pages
  • Select branch: main
  • Save
⬇️
Result: Live link generated!

https://username.github.io/repository-name

This is your online identity!

PART 5 – Domain & DNS

What is a Domain?

Examples: google.com, github.com

Domain = Human-readable website name

What is DNS?

DNS = Domain Name System

Converts: google.com → 142.250.183.14 (IP Address)

Think of it like a phone contact:

Domain = Contact Name

IP Address = Phone Number

Domain Purchasing Platforms

Popular Domain Providers:

  • GoDaddy
  • Namecheap
  • Google Domains

Connecting Custom Domain to GitHub Pages

Conceptual Overview:

  1. Buy domain from provider
  2. Go to DNS settings
  3. Add A record pointing to GitHub IP
  4. Add CNAME file in repository

(Conceptual demo only - no deep technical configuration needed)

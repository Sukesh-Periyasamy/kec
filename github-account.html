GitHub Account Creation Guide

Complete Student Guide to GitHub

PART 1 – What is GitHub?

GitHub

GitHub is a cloud platform where developers:

  • Store code online
  • Collaborate with others
  • Track versions and changes
  • Build professional portfolio
  • Contribute to open-source projects

Tell Students:

Your GitHub profile = Your Coding Resume

Recruiters check GitHub profiles before interviews.

PART 2 – How to Create GitHub Account

Step-by-Step Process

Open Official Website

https://github.com

Click Sign up

Enter Email Address

Use a professional email:

Good Examples

  • sukesh.dev@gmail.com
  • john.smith@outlook.com
  • student.name@college.edu

Avoid

  • coolboy123@gmail.com
  • gamer456@yahoo.com
  • xyz_fun@hotmail.com

Create Strong Password

Minimum requirements:

  • 8+ characters
  • Include uppercase letters
  • Include numbers
  • Include symbols (@, #, $, etc.)

Choose Username (VERY IMPORTANT)

This becomes your public URL:

https://github.com/yourusername

Choose a professional username:

Good

  • sukesh-dev
  • sukesh-ml
  • john-smith-dev

Avoid

  • king_sukesh_007
  • coolguy123
  • xX_gamer_Xx

Warning: Username cannot be changed easily later!

Email Verification

Check your inbox → Click verification link

Choose Plan

Select:

Free Plan (More than enough for students)

Account Created Successfully!

PART 3 – Understanding GitHub Dashboard

After login, students will see:

  • Repositories
  • Profile
  • Projects
  • Issues
  • Pull Requests

Important Sections Students Must Learn

Profile Page (MOST IMPORTANT)

Click your profile photo → Your profile

This page shows:

  • Your bio and introduction
  • Your repositories
  • Contribution graph (green squares)
  • Followers and following

Students Should:

  • ✔ Add profile photo
  • ✔ Write professional bio
  • ✔ Add college name
  • ✔ List key skills
  • ✔ Add LinkedIn profile link

Repositories

Repository = Project folder stored online

To create a repository:

  1. Click New
  2. Enter repository name
  3. Select Public
  4. Check "Add a README file"
  5. Click Create repository

Repository Naming Examples

Good Names
  • portfolio-website
  • student-management-system
  • iot-temperature-monitor
Bad Names
  • project1
  • test123
  • finalfinalnew

Repository Structure

Inside a repository, you'll find:

  • Code files and folders
  • README.md (project documentation)
  • Commit history
  • Branches
  • Settings

What is README.md?

This is the first thing recruiters see when visiting your repository!

Students should write:

  • Project title
  • Description of what it does
  • Technologies used
  • How to run the project
  • Screenshots or demos

Branches

Students must understand:

  • main branch - The stable, production version
  • feature branches - For development work

Used for:

  • Team development
  • Safe updates without breaking main code

Commits

Commit = Save point in your project

Students should write meaningful commit messages:

Good Commit Messages

  • "Added login page UI"
  • "Fixed navbar responsive design"
  • "Implemented user authentication"

Bad Commit Messages

  • "update"
  • "changes"
  • "asdf"

PART 4 – How Students Will Use GitHub

Upload Projects

  • After learning HTML/CSS → Push portfolio project
  • After IoT workshop → Push ESP32 project
  • After ML course → Push ML code

Portfolio Building

Share your GitHub link:

https://github.com/username

This increases internship chances!

GitHub Pages (Free Website Hosting)

Students can host portfolio websites for free!

URL format:

https://username.github.io/repository-name

Important GitHub Features

Fork

Copy someone's project to your account.

Used in open-source contribution.

Pull Request

Request to merge your changes into a project.

Used in team development and open-source.

Issues

Used to:

  • Report bugs
  • Suggest new features
  • Track tasks

Contribution Graph

Green boxes show your daily activity.

More green = More consistency

Tell students: Try to commit code daily for 30 days!

Important Settings Students Must Configure

Profile Settings

Go to: Settings → Profile

Add:

  • Professional bio
  • Location (City, Country)
  • Personal website (if any)
  • LinkedIn profile URL
  • Twitter/X handle (optional)

Email Settings

Go to: Settings → Emails

Enable:

  • ✔ Keep my email address private

SSH and GPG Keys

Go to: Settings → SSH and GPG Keys

(Advanced – optional for beginners)

Final Tips for Students

  • Keep code organized and well-documented
  • Write README files for every project
  • Commit regularly with meaningful messages
  • Keep your profile updated
  • Contribute to open-source projects
  • Build a consistent contribution streak

1 Experiment Details

Experiment No: 04
Experiment Name: Cloud-Based Temperature and Humidity Monitoring using ESP32
Duration: 3 Hours

2 Aim

To design and implement a cloud-based IoT system that reads temperature and humidity using DHT11 sensor and displays data on a live web dashboard.

3 Learning Outcomes

After completing this experiment, students will be able to:

  • Understand microcontroller-based IoT architecture
  • Interface DHT11 sensor with ESP32
  • Connect ESP32 to WiFi
  • Send sensor data to cloud server
  • Create and host a live dashboard using GitHub

5 Theory

5.1 What is IoT?

Internet of Things (IoT) is a system where physical devices collect and exchange data through the internet.

5.2 System Architecture

Sensor → ESP32 → WiFi → Cloud → Website

Explanation:

  • Sensor measures environment
  • ESP32 processes data
  • WiFi sends data to cloud
  • Cloud stores data
  • Website displays data

5.3 About ESP32

ESP32 is a WiFi and Bluetooth-enabled microcontroller used for IoT applications.

Features:

  • Built-in WiFi
  • Multiple GPIO pins
  • Low power consumption
  • Supports HTTP communication

5.4 About ThingSpeak

ThingSpeak is a cloud-based IoT platform that stores and visualizes sensor data.

Features:

  • Free student version
  • Auto graph generation
  • API-based data update
  • CSV export

5.5 About GitHub Pages

GitHub Pages is a free static website hosting service used to host the IoT dashboard.

7 Procedure

Step 1 – Install Software

  • Install Arduino IDE
  • Add ESP32 board support
  • Install DHT library

Step 2 – Hardware Setup

  • Connect DHT11 to ESP32
  • Verify wiring connections

Step 3 – Sensor Testing

  • Upload basic DHT reading code
  • Open Serial Monitor
  • Confirm temperature values

Step 4 – Connect to WiFi

  • Add WiFi credentials
  • Upload code
  • Verify WiFi connection in Serial Monitor

Step 5 – Configure ThingSpeak

  1. Create new channel
  2. Add fields
  3. Copy Write API Key
  4. Update ESP32 code
Verify: Graph updates every 15 seconds.

Step 6 – Create GitHub Dashboard

  1. Create GitHub repository
  2. Upload index.html
  3. Replace Channel ID and Read API Key
  4. Enable GitHub Pages
  5. Open public dashboard URL
